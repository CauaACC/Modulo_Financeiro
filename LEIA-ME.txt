CONFIGURAÇÃO:
-------------
1. Ajuste as credenciais do banco em:
   domain/ADODBConnection.php
   (host, user, pass, db)

2. Ajuste o COD_EVENTO padrão em:
   include/config.php (linha: define('COD_EVENTO', '115'))

3. Ajuste WB_PATH em include/config.php se o sistema
   não estiver na raiz /sistema/ do servidor web.

4. Configure o servidor web (Apache/Nginx) para
   apontar o document_root corretamente.


TABELAS DO BANCO DE DADOS NECESSÁRIAS:
--------------------------------------
Tabelas do módulo financeiro:
  - fin_lancamentos_contabeis
  - fin_plano_contas
  - fin_centro_custo
  - fin_rateios
  - fin_fontes_recurso

Tabelas de autenticação/permissões:
  - sis_usuarios
  - sis_grupos
  - sis_ligacao_usuarios_grupos
  - sis_ligacao_usuarios_eventos
  - sis_direitos_grupos_sistemas
  - sis_sistemas
  - tab_convidados (opcional, para nome de exibição)
  - sis_gestor (referenciada em JOINs mas não obrigatória)

A permissão financeira usa: SIS_CONTROLE_FINANCEIRO = 7
O usuário precisa estar em um grupo com cod_sistema=7 e cod_direito=5.


COMO FUNCIONA:
--------------
- Login via sis_usuarios (bcrypt)
- Verifica permissão SIS_CONTROLE_FINANCEIRO
- Todas as operações CRUD via AJAX (jQuery + CommandAC.php)
- Roteamento: ?c=NomeClasse&m=metodo → controller/NomeClasseAC.php
- VOs usam magic methods (__call) para get/set automáticos
- DAOs usam ADODB com GetInsertSQL/GetUpdateSQL para SQL dinâmico
